利用Freenom + CloudFlare + Github Pages搭建个人博客

Leiyiyan 2020-08-16 GitHub Pages

  • Freenom：是世界上第一个也是唯一的免费域名提供商。

  • CloudFlare：以向客户提供网站安全管理、性能优化及相关的技术支持为主要业务。

  • GitHub Pages：是 Github 提供的一种免费的静态网页托管服务。

# 前言

拥有一个自己的博客, 是每个程序员的梦想。个人博客的搭建方式也是各种各样，如果你只是想单纯做一个博客，和世界分享你的观点和视角，那么我推荐使用 GitHub Pages，接下来开始进入主题，利用Freenom + CloudFlare + GitHub Pages（“白嫖使我快乐”三件套）开始搭建个性化域名的个人博客网站。

# Github申请

  1. 前往GitHub官网，快速申请我们的账号。

  2. 完成账号注册后，GitHub会对我们的账号进行检测，检测完成点击下方的“Join a free plan”按钮。

  3. 这里会让我们选择或填写个人的相关信息，我们可以不选择填写，直接下方的“Complete setup”按钮。

  4. 最后一步，GitHub会给我们注册的邮箱里发送验证邮件，我们前往邮箱进行验证即可。

  5. 到此，我们的GitHub账号已经注册完成，我们点击“Create a repository”按钮，创建我们的仓库。

  6. 创建仓库这里我们需要重点注意，因为我们需要创建GitHub提供的免费静态网页站点，为节省不必要的时间，仓库名必须是“Giuhub用户名.github.io”，其他信息可不填写，点击下方的“Create repository”。

  7. 接下来，我们为我们的博客网站创建一个首页，点击“creating a new file”。

  8. 我们创建一个“index.html”，这里我们随便编写一下首页代码，后续我们可通过相关的博客框架进行开发。

    <html>
    <head>
        <title>My Blog</title>
    </head>
    <body>
        <h1>This is my blog</h1>
    </body>
</html>

  9. 首页创建完成后，我们点击“Settings”按钮，查看我们的GitHub Pages是否正常启用。

  10. 在打开的设置页面，我们拖到下方的“GitHub Pages”选项，如图所示即表明我们的GitHub Pages已经可以正常访问

  11. 点击我们的GitHub Pages访问地址，成功打开之前编辑的首页内容。

到此，我们的GitHub Pages就已经注册完毕，如果我们想使用自己的个性化域名，而不想使用GitHub提供给我们的地址访问我们的网站呢？别急，精彩继续！

# Freenom域名申请

  1. 前往Freenom官网，在首页输入你想要的域名，点击右侧的“检查可用性”。

  2. 目前Freenom支持4种（.tk、.ml、.ga、.cf）免费域名的申请，选中自己喜欢的域名，点击右侧的“现在获取”，右上方出现“1购物车中的域”时，点击“完成”。

  3. 在右侧选择域名的有效期，Freenom可支持最长12个月的免费使用期限，我们那必须选择12个月咯（“白嫖”使我快乐）。

  4. 接下来Freenom需要我们登录账号，我们可以在左侧使用自己的邮箱进行注册，也可选择右侧的Google、Facebook账号登录注册。

  5. 在完成邮箱验证后，我们需要填写我们的账号信息，这里我们可以在M国身份大全网站中查找身份信息，并填写到Freenom详细信心栏中，需要注意：电话号码需要删除“-”，否则会报错。在信息填写完并确认无误后，勾选下方“同意条款和条件”，并点击右侧的“Complete Order”按钮。

  6. 当显示订单信息后，我们申请的个性化域名就可以免费使用了。

# CloudFlare域名解析

  1. 前往CloudFlare官网注册账号

  2. 完成账号的申请及邮箱的验证。

  3. 回到首页，我们点击“Add Site”按钮，添加站点。

  4. 这里我们选择免费的服务即可，点击下方的“confirm plan”。

  5. 这里我们需要对我们之前申请的GitHub Pages的域名及IP进行解析！其中blogdemo.tk的内容部分为我们在终端上ping我们的GitHub Pages的IP地址，如：ping blogdemo.github.io。

  6. 接下来CloudFlare会提供给我们两个“Nameserver”，我们需要到我们的域名提供商那里进行“Nameservers”的修改。

  7. 我们返回Freenom，登录我们申请域名的账号，点击上方的“Services”菜单中的“My Domains”选项，下方会显示我们已注册的域名信息。此时，我们点击右侧的“Manage Domain”按钮，管理我们的域名。

  8. 我们点击“Management Tools”，选择“Nameservers”选项。

  9. 这里我们选择“Use custom nameservers (enter below)”选项，并输入第6步中CloudFlare提供给我们的两个“Nameserver”，填写完成后，点击下方的“Change Nameservers”。

  10. 此时我们回到CloudFlare，点击下方的“Done, check nameservers”，或者“Re-check now”按钮进行“Nameservers”的验证。

  11. 验证成功后，我们回到首页，看到我们的站点状态已经生效。

  12. 接下来，我们配置HTTPS，我们点击我们的站点，选择上方的“SSL/TLS”按钮，加密模式我们选择“完全”。

  13. 切换的“边缘证书”选项，开启下方的“始终使用HTTPS”选项。

# GitHub域名绑定

  1. 我们回到GitHub，在“Custom domain”中输入我的个性化域名，点击“save”保存，并开启下方的“Enforce HTTPS”，GitHub将开始对域名进行绑定，此过程耗费时间较长，我们可以喝杯茶，吃个点心，安心等待即可。

  2. 待GitHub绑定域名完成后，我们在浏览器中输入我们的个性化域名地址，如果能成功打开首页内容，那么恭喜你，此刻开始，你已经入坑博客主了，开启你的博客生涯吧！

# 结尾

到此，我们的博客已经完成初始化，此处应该应该有掌声！接下来我们可以使用VuePress、Hexo等框架来快速完成博客的搭建了。
